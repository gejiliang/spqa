## Senatus PopulusQue Agentium

**元老院与 Agent · AI-Native Organization Architecture**

> 动态组阁 · 四阶段议制 · 三级生命周期

**官网 Official Website: [idiotstudio.com/spqa](https://idiotstudio.com/spqa/)**

SPQA 是基于 [OpenClaw](https://github.com/gejiliang) 构建的 AI-Native 组织架构范式，以罗马共和国政治体制为蓝本，为多 Agent 协作提供一套完整的编排、决策与执行框架。

SPQA is an AI-Native organizational architecture built on **OpenClaw**, inspired by the Roman Republic's governance system. It provides a complete orchestration, deliberation, and execution framework for multi-agent collaboration.

## 元老院四阶段议制 · Ordo Deliberandi

| Phase | 阶段 | 方法 Method | 拉丁名 Latin |
|-------|------|-----------|-------------|
| 1 | 发散期 | 六顶思考帽 | Cogitatio |
| 2 | 冲突期 | 红蓝对抗 | Contentio |
| 3 | 收敛期 | 德尔菲法 | Consensus |
| 4 | 产出期 | 蓝帽总结 | Decretum |

**议事纪律 · Lex Senatus**

- 若无新增价值或实质性反对，禁止表达附和（Silentium Rule）
- 每轮发言需标注视角维度（Perspectiva Tag）
- 超过 3 轮未收敛，提交凯撒裁决（Appellatio ad Caesarem）
- 产出必须包含：军团兵角色清单 + 分工 + 交付标准（Decretum Format）

## 禁卫军 · Praetorian Guard

凯撒直属持久化项目组 · 独立于元老院指挥链

**标准角色骨架（可按业务增减）：**

| 角色 Role | 职责 |
|-----------|------|
| Strategus · 谋略官 | 策略制定 · 方向把控 |
| Custos · 守卫官 | 风控管理 · 边界守护 |
| Executor · 执行官 | 核心任务执行 |
| Tabularius · 记录官 | 数据维护 · 状态追踪 |

**禁卫军条令 · Lex Praetoriana**

- 每队全员持久化，项目存续期内 7×24 运行
- 凯撒直接指挥，不经执政官和元老院
- 可根据业务需要增减队内角色（标准四角色为骨架）
- 成果由史官（Annalist）归档，定期向凯撒汇报

## 任务流转 · Iter Mandati

```
Caesar decernit（凯撒下达议题）
        ↓
Consul & Praetor（执政官接令 + 裁判官分析组阁）
        ↓
Senatus deliberat（元老院四阶段议制）
        ↓
Senatus Consultum（产出元老院决议：角色 + 分工 + 交付标准）
        ↓
Consul legiones scribit（执政官征召军团兵）
        ↓
Legionarii exsequuntur（军团兵执行 → 交付）
        ↓
Annalista scribit（史官归档）
```

**Via Praetoriana · 禁卫军独立流转：**

```
Caesar decernit → Cohors exsequitur → Annalista scribit
（不经元老院 · 凯撒直接指挥 · 实时性优先）
```
